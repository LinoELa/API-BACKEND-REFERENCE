# API Backend Reference - Contexto del Proyecto

## Resumen

Este proyecto no es un template de API ni una base de backend para ejecutar.
Es una referencia de estudio construida a partir de un PDF fuente sobre conceptos de API y backend.

La meta principal es transformar ese material original en apuntes Markdown mas claros, ordenados y faciles de revisar.

## Documento fuente

El origen del contenido esta en:

- `00-HandBook/API-ALL-CONCEPTS.pdf`

Ese PDF actua como handbook base.
Los archivos `.md` del repositorio son una adaptacion pedagogica del contenido, no una transcripcion exacta.

## Estructura del repositorio

- `00-HandBook/`
  - PDF completo
  - `handbook.md` con contexto general
- `01-BASIC-API-CONCEPT/`
  - `Q1-Q30`
- `02-INTERMEDIATE-API-CONCEPTS/`
  - `Q31-Q60`
- `03-ADVANCED-REAL-TIME-SCENARIOS/`
  - `Q61-Q100`

## Formato esperado del contenido

Cada seccion debe mantener:

- un `00-indice.md`
- archivos partidos por bloques de preguntas
- titulos limpios
- explicaciones en espanol
- terminos tecnicos en ingles cuando ayuden
- ejemplos breves y practicos

## Intencion del proyecto

Este repositorio sirve para:

- estudiar
- repasar
- preparar entrevistas
- consultar conceptos rapidamente

No debe tratarse como:

- boilerplate backend
- template de arquitectura
- API productiva lista para desplegar

## Regla editorial

Si se sigue ampliando el contenido, lo correcto es:

- respetar la estructura por secciones
- actualizar indices
- mantener consistencia de nombres
- priorizar claridad por encima de copiar literalmente el PDF
